About the program

This is a console terminal based application that allows the user to record the games they have played. The app is entirely terminal based and is navigated through prompts.

Features

Users can create an account to access their library. When adding a new game to the library, the user will be asked for the following :

  • Name
  • Console
  • Year of release
  • Genre
  • Rating (out of 10)
  • Completed (yes or no)
  • Start Date
  • Finish Date
  • Time Played
  • Comment

Once the user has filled in the details, the database will be updated with the new entry.

Visualising

Users can view their library listing through console commands. The application uses the "ConsoleTables" package to display the results of the user's searches.

Connection & Databases

This game library uses databases to hold user data, hosted on Railway. When opening the app, users can sign up, or log in. Passwords are hash encrypted and salted using the BCrypt .Net package.

Why I made this

This applicaton combines two interests of mine : Archiving and Retro technology aesthetics. I wanted to create a program to track the games I own and have played, including information I find interesting such as when I played the games and for how long. I enjoy the simple interface of typing out the details and queries. I also thought this would combine well with my CRT desktop monitor.
